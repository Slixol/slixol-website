  <action>
The current code in Footer.tsx already has `overscrollCount.current >= 3` logic (lines 66-87). The client says it triggers on first attempt. Diagnose and fix:

**Root cause:** The `passive: true` event listener is correct. The real issue is that a single trackpad gesture often fires many rapid `wheel` events with `deltaY > 0` — one "attempt" on a trackpad = dozens of events, causing the counter to hit 3 within a single physical gesture.

**Fix:** Add a timestamp-based debounce so each "attempt" is counted once per distinct gesture, not once per wheel event. Replace the current useEffect content:

```typescript
useEffect(() => {
  let lastAttemptTime = 0;

  function handleWheel(e: WheelEvent) {
    const atBottom =
      window.innerHeight + window.scrollY >= document.body.scrollHeight - 2;

    if (atBottom && e.deltaY > 0) {
      const now = Date.now();
      // Count one attempt per gesture (debounce: 400ms between attempts)
      if (now - lastAttemptTime > 400) {
        lastAttemptTime = now;
        overscrollCount.current++;
        if (overscrollCount.current >= 3) {
          overscrollCount.current = 0;
          document.getElementById("konzultacio")?.scrollIntoView({ behavior: "smooth" });
        }
      }
    } else if (!atBottom) {
      // Reset counter if user scrolls away from bottom
      overscrollCount.current = 0;
    }
  }

  window.addEventListener("wheel", handleWheel, { passive: true });
  return () => window.removeEventListener("wheel", handleWheel);
}, []);
```

Key changes from original:
- `lastAttemptTime` local variable (not ref — only needed within this closure)
- 400ms debounce between counted attempts — one trackpad swipe = one attempt
- Reset only fires when `!atBottom` (not on every non-overscroll event) to avoid resetting during scroll momentum
